# LLM Wiki Schema (지식베이스 운영 스키마)

본 문서는 `my-wonderful-workbase/wiki` 디렉토리를 지식베이스로 관리하기 위한 기준과 규칙을 담고 있습니다. LLM 에이전트는 이 스키마에 따라 위키를 읽고, 쓰고, 유지보수합니다.

## 디렉토리 구조
- `raw/assets/`: 원본 문서들이 위치하는 곳입니다. (예: PRD, SRS, 리서치 문서 등) **LLM은 이 디렉토리의 파일들을 읽기만 하며, 절대 수정하지 않습니다.**
- `wiki/sources/`: `raw/assets/` 의 원본 문서를 수집(Ingest)하여 요약, 주요 핵심 내용, 연결된 개념/엔티티를 기록한 마크다운 파일들의 모음입니다.
- `wiki/concepts/`: 원본 문서들을 통해 도출된 핵심 개념, 비즈니스 모델, 도메인 지식 등에 대한 설명 문서입니다.
- `wiki/entities/`: 특정 인물, 조직, 제품, 도구 등 고유명사와 관련된 상세 정보를 담고 있는 문서입니다.
- `wiki/index.md`: 위키 전체 문서들의 카탈로그(목록)입니다. 문서를 새로 생성할 때마다 이곳에 링크를 업데이트해야 합니다.
- `wiki/log.md`: 수집, 질의, 린트 등의 작업 내용을 연대기순으로 기록하는 일지입니다.

## 핵심 운영 워크플로우

### 1. Ingest (수집)
새로운 원본 문서가 추가되거나 사용자가 "문서 [X]를 Ingest 해줘"라고 요청할 시 다음 절차를 따릅니다.
1. `raw/assets/` 경로의 원본 문서를 꼼꼼히 읽고 분석합니다.
2. `wiki/sources/[문서명].md` 파일을 생성하여 요약, 주요 시사점, 관련 엔티티/기존 개념들을 정리합니다.
3. 분석 과정에서 새롭게 발견되었거나 업데이트가 필요한 개념/엔티티가 있다면 `wiki/concepts/` 또는 `wiki/entities/` 아래의 관련 문서를 생성하거나 수정합니다.
4. `wiki/index.md`를 열고 새롭게 생성된 파일 또는 업데이트된 파일의 링크와 짧은 설명을 업데이트합니다.
5. `wiki/log.md`에 다음과 같은 형식으로 작업 기록을 추가합니다.
   - `## [YYYY-MM-DD] ingest | 원본 파일명`

### 2. Query (질의)
사용자가 현재 지식베이스와 관련된 질문을 할 경우 다음 절차를 따릅니다.
1. 먼저 `wiki/index.md` 또는 전체 문서 목록을 확인하여 질문과 연관된 페이지들을 탐색합니다.
2. 관련 개념, 엔티티, 소스 페이지를 종합적으로 읽은 후 답변을 생성합니다. (답변에는 반드시 출처를 인용해야 합니다)
3. 만약 도출된 내용이 새롭고 가치있는 통찰이거나 비교 분석이라면, 일회성 답변으로 끝내지 않고 `wiki/concepts/` 등에 새로운 페이지로 저장한 뒤 `wiki/index.md`와 `wiki/log.md`를 업데이트할 것을 사용자에게 제안하거나 직접 수행합니다.
4. 작업 완료 후 `wiki/log.md`에 다음과 같은 형식으로 기록합니다.
   - `## [YYYY-MM-DD] query | 질의 내용 요약`

### 3. Lint (점검 및 최적화)
사용자가 위키 상태 점검 (Lint)을 요청할 경우 다음을 수행합니다.
1. 고립된 페이지(인바운드 링크가 없는 페이지) 색출
2. 서로 상충되는 주장이나 모순이 있는 내용 플래깅 및 사용자에게 보고
3. 존재하지 않는 링크(Missing cross-references) 점검
4. `wiki/log.md`에 다음과 같은 형식으로 기록합니다.
   - `## [YYYY-MM-DD] lint | 수행한 린트 내용 요약`

## 문서 작성 컨벤션
* 전면부(Frontmatter): 가급적 파일 최상단에 YAML 포맷으로 태그(`tags:`), 생성일(`date:`), 관련 원본 출처(`sources:`) 등을 기입합니다.
* 교차 참조(Cross-reference): 다른 위키 페이지를 언급할 때는 반드시 절대/상대 경로 혹은 마크다운 링크 문법(`[페이지명](../개념/페이지명.md)`)을 사용하여 연결합니다.
